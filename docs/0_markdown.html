Programming Languages and Their Working Environments

Before we start, it is important to understand the difference between the programming language itself and the program in which we write.

1. The Language: R and Python

R and Python are the “engines” behind your analysis. For your computer to understand what you want from it, you need to download the language as base software.

  • Python: Installed as a program from python.org.
  • R: Installed as “Base R” from the CRAN network.

Important: You will almost never open these programs directly. They “rest” in the background of your operating system and wait for commands.

2. The Tool: The IDE (RStudio & VS Code)

To actually write code, we use an IDE (Integrated Development Environment). This is your actual control center.

  • RStudio is the standard environment for R.
  • VS Code is the most popular environment for Python.

What Do You Need for Setup?

If you later want to work on your own computer, this is the correct order:

  1. Install the engine: Download Python or R (provides the “language”).
  2. Set up the workspace: Install an IDE like RStudio or VS Code (your workstation).
  3. Unpack the tools: Install so-called “packages” within the IDE. These are extensions for specific tasks, such as drawing biological diagrams.
TipOur Tip for Getting Started

To work on the exercises on this website, no installed RStudio or VS Code is required. Our interactive windows already have everything “on board”. However, we recommend that you familiarize yourself with these programs during the course, as they will be your daily companions in later practice!

Working with Markdown

In this course we do not use ordinary text documents or pure program code. In addition to this website, we work with Markdown files as an offline alternative.

In science, reproducibility (traceability) is crucial. Markdown allows us to do so-called Literate Programming: * We write our text (explanations, hypotheses). * We embed the actual code (R or Python) directly within it. * We let the computer render the document (in RStudio often via the Knit button).

The result is a finished document (HTML, PDF, or Word) in which the text, the code, and the resulting graphics are perfectly combined.

The Structure of a File

  1. The text: Regular text with simple formatting (**bold**, *italic*, # Heading).
  2. The code chunks: Blocks that begin with three backticks. This is where the actual code is executed. It is recommended to name each chunk — this keeps your document tidy and greatly facilitates troubleshooting.
  3. The YAML header: The area at the very top between the --- lines for title and author.

Focus: R and RStudio

R is a powerful language for statistical data analysis. We recommend that you also take the introductory courses at DataCamp to get a feel for the syntax. RStudio is an “Integrated Development Environment” (IDE) that helps you write code, visualize data, and save results.

The RStudio Interface

The interface is divided into four main areas:

  1. Source (top left): This is where you write your scripts and Markdown files. To create a new file, click on the New File icon (green plus) at the top left. Everything written here can be saved.
  2. Console (bottom left): Code is executed immediately here. For example, type 3 + 4 and press Enter — the result appears immediately. The console is ideal for commands that you do not need to save permanently.
  3. Environment (top right): All variables, lists, and datasets that you have created and defined during your session are displayed here.
  4. Files/Plots/Help (bottom right): Here you manage your files, view generated graphics, or search the help for functions (e.g., with ?matrix).

First Steps in Programming

  • Assignments: Values are loaded into variables with <- or = (e.g., a <- 10). The variable then appears in the Environment.
  • R as a calculator: R is excellent for mathematical calculations. You can calculate directly with variables:
#| standalone: true
#| viewerHeight: 300
library(shiny)
ui <- fluidPage(
  textAreaInput("code", "R Chunk: Mathematical Operations", value = "x <- 5\ny <- 3\nSum <- x + y\nSum * 5", width = '100%', height = '100px'),
  actionButton("run", "Run Code", class = "btn-success"),
  verbatimTextOutput("out")
)
server <- function(input, output) {
  observeEvent(input$run, { output$out <- renderPrint({ eval(parse(text = input$code)) }) })
}
shinyApp(ui, server)
  • Case Sensitivity: R is extremely picky. hello is a different variable from Hello.
  • Getting help: Use the documentation directly in RStudio or online resources:
    • guru99 R Tutorial
    • Babraham Bioinformatics
    • R for Data Science
  • Workspace: Use ls() to list your variables and rm(a) to delete them — Caution: rm() deletes irreversibly, so use it carefully! Your entire environment can be saved as an .RData file and restored in a later session with load("MyEnvironment.RData").
  • Working Directory: Via the Files -> More -> Set As Working Directory tab, you define which folder R should look in for your files.

Helpful Tips in the Editor

  • Autocomplete: In RStudio, quotation marks "" and brackets () usually appear automatically in pairs. This helps you avoid syntax errors caused by forgotten characters.
  • History: Using the arrow keys (up/down) you can scroll through your most recently used commands in the console.
  • Comments: Everything after a # is ignored by R. Use this generously to explain your code to yourself and others.

Python and VS Code

Python is a versatile and widely used programming language that is particularly well-suited for data analysis, machine learning, and scientific computing. We recommend that you also take the introductory courses at DataCamp to get a feel for the syntax. VS Code is a flexible development environment (IDE) that can be optimally adapted for Python through extensions.

The VS Code Interface

The interface is divided into four main areas:

  1. Editor (center): This is where you write your scripts and notebooks. To create a new file, click on the New File icon at the top left in the Explorer. Everything written here can be saved. The file extensions determine in which programming language you are writing and how the file can be used (.py/.ipynb for Python, .r for R, .md for Markdown).
  2. Terminal (bottom): Code is executed immediately here and files can be launched. Connections to clusters, communication with GitHub, or searching for files also work via the terminal. The terminal can be opened, for example, at the top right (3rd icon from the right).
  3. Explorer (left): Here you manage your files and folders. These are either stored locally on your computer or on a cluster.
  4. Control Center (far left): Here you can, for example, search documents for a keyword, connect and manage GitHub repositories, debug, install extensions, and manage remote connections (e.g., clusters).

First Steps in Programming

  • Assignments: In Python, values are loaded into variables with = (e.g., a = 10). Unlike R, there is no <- in Python.
  • Python as a calculator: Python is excellent for mathematical calculations. You can calculate directly with variables:
#| standalone: true
#| viewerHeight: 300
from shiny import App, ui, render, reactive
import io, contextlib

app_ui = ui.page_fluid(
    ui.input_text_area("code", "Python Chunk: Mathematical Operations", value="x = 5\ny = 3\nSum = x + y\nprint(Sum * 5)", width='100%', height='100px'),
    ui.input_action_button("run", "Run Code", class_="btn-success"),
    ui.output_text_verbatim("out")
)

def server(input, output, session):
    @render.text
    @reactive.event(input.run)
    def out():
        f = io.StringIO()
        with contextlib.redirect_stdout(f):
            exec(input.code(), {})
        return f.getvalue()

app = App(app_ui, server)
  • Case Sensitivity: Python is just as picky as R. hello is a different variable from Hello.

Getting Help:

Use the documentation directly in VS Code or online resources: Python Documentation W3Schools Python Tutorial Python for Data Science Handbook

  • Workspace: Use dir() to list all current variables and objects, and del a to delete a variable — Caution: del deletes irreversibly, so use it carefully!

  • Working Directory: With import os and os.getcwd() you can display the current directory; with os.chdir("path/to/folder") you change it.

Helpful Tips in the Editor

  • Autocomplete: VS Code automatically suggests functions, variable names, and brackets. Quotation marks "" and brackets () often appear automatically in pairs — this helps avoid syntax errors.
  • History: In the terminal you can scroll through your most recently used commands using the arrow keys (up/down).
  • Comments: Everything after a # is ignored by Python — just like in R. Use comments generously to explain your code.
